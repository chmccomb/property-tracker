# JC Heights Property Analysis

A real estate market analysis tool for Jersey City Heights, built on Paragon MLS data (Hudson County).

## Project Structure

```
jc-heights-analysis/
├── data/
│   ├── raw/                     # Original Paragon MLS exports (do not edit)
│   │   ├── MLS_export.csv       # Full MLS transaction export (51-column Paragon format)
│   │   ├── Market_Condition_Detail_dom.csv
│   │   └── CMA_Spreadsheet.csv
│   └── processed/               # Generated by scripts — can be regenerated
│       ├── jc_heights_cleaned.csv
│       ├── jc_heights_scored.csv
│       ├── jc_heights_block.csv
│       ├── jc_heights_blocks_v2.csv
│       ├── dashboard_data.json
│       └── dashboard_data_v2.json
├── scripts/
│   ├── jc_heights_clean.py      # Step 1: cleans raw MLS → jc_heights_cleaned.csv
│   └── jc_heights_emerging.py   # Step 2: scores blocks → dashboard_data_v2.json
├── dashboard/
│   ├── jc_heights_dashboard_v2.html  # Main deliverable (open in browser, no server needed)
│   └── jc_heights_dashboard_v1.html  # Prior version
└── README.md
```

## How to Refresh Data

When you have a new Paragon export, replace `data/raw/MLS_export.csv` and run:

```bash
cd scripts

# Step 1 — clean & normalize MLS data
python jc_heights_clean.py

# Step 2 — score blocks, build dashboard JSON
python jc_heights_emerging.py
```

The scripts output to `data/processed/`. The dashboard HTML embeds `dashboard_data_v2.json` inline — after re-running, you'll need to rebuild the HTML (see note below).

## Dashboard

Open `dashboard/jc_heights_dashboard_v2.html` directly in a browser — no server required. The JSON data is embedded inline in the file.

**Tabs:**
- **Map** — color-coded block layer (green/yellow/red) by deal score; click blocks for details
- **Deal Scorer** — score any property by address or enter custom specs; includes Fair Price Calculator
- **Trends** — block-level PSF trends over time

## Key Scoring Concepts

### Block Quality Score
- **Established score** (backward-looking): CAGR 30% + DOM 20% + STL 20% + Assessment ratio 15% + Inventory 15%
- **Emerging score** (forward-looking): Undervaluation 30% + Acceleration 30% + DOM tightening 20% + Demand 20%
- **Combined**: `est * 0.35 + emg * 0.65` (emerging-weighted for appreciation focus)

### Deal Score
`deal = blockQuality * 0.60 + priceScore * 0.40`

Price score compares the listing's PSF to an amenity-adjusted block median:
- Parking premium: +2% (1BR), +14.2% (2BR), +3.4% (3BR)
- Outdoor space premium: +4.3%
- Size curve adjustment (PSF varies ~27% by sqft within bed type)
- HOA capitalization at 7% discount rate: `monthly_hoa * 12 / 0.07`

### Appreciation
Sold comps are forward-projected to today using the block's CAGR:
`todayPsf = soldPsf * (1 + CAGR) ^ yearsSinceSale`

### Known Limitations
- **Tax block ≠ building**: Jersey City tax blocks span multiple addresses/buildings. Block 2801 (e.g.) includes 4 different streets. Blocks with `n_sales < 10` show a reliability warning.
- **Score compression**: Sub-signals are min-max normalized 0–1 then averaged, so composite scores naturally cluster in the middle. Re-normalization applied post-combination.
- **Bed-matched fallback**: Some properties fall back to all-bed block median when same-bed comp count is low (< 3 in 24mo, < 5 in 36mo).

## Data Notes

- Source: Paragon MLS, Hudson County (Jersey City Heights subset)
- Coverage: Closed sales + active listings
- Amenity premiums derived from hedonic regression on JC Heights closed sales
- CAGR calculated from rolling 12-month median PSF by block

## Updating the Dashboard HTML

After re-running scripts, paste the new `dashboard_data_v2.json` into the dashboard HTML by replacing the embedded JSON in the `<script id="dashboard-data">` tag, or ask Claude to rebuild the dashboard file with the new data.
